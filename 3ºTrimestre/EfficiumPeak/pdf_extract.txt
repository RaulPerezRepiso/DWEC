Actividades propuestas para el periodo FEOE 25/26 para el 
módulo de Desarrollo web en entorno cliente (2º GS DAW) 
 
Actividad 
El alumno/a debe desarrollar un proyecto web que 
versará sobre un tema de libre elección o que tenga 
algún interés práctico para la empresa en la que 
realiza la FEOE: recursos humanos, inventariado 
de material, facturación, réplica de las aplicaciones 
que usan en la empresa en modo aplicación web, 
etc..  
Requisitos: 
 
● RA2. c), e), f) Se valorará el uso de 
comentarios, código legible, limpio y modular, 
así como el uso apropiado de estructuras de 
control y funciones. 
● RA3. c), e), f), g)  Uso de objetos nativos de 
JavaScript (Date, Array, String, Number, 
Window, Navigator, Location, …) y de sus 
métodos. También se implementarán objetos 
y/o arrays de objetos con sus 
correspondientes propiedades. 
 
Se usarán ventanas auxiliares para mostrar 
más información al usuario, mostrar otras 
partes de la aplicación cuando sean 
necesarias, etc. 
 
RA4. d), g), h) Se implementarán clases para 
manejar “entidades” que se crean necesarias: 
usuarios, artículos, 
 
● RA5. d), f)  Uso de, al menos, tres eventos 
distintos ( por ejemplo, tres eventos click se 
considera un solo tipo de evento ). Los eventos 
se asignarán en el código JavaScript y no en 
HTML. 
 
● RA6. d), e) Creación, modificación, y 
eliminación dinámica de nodos DOM. 
 
Acceso y modificación de los atributos y a las 
propiedades de estilo (CSS3) de los 
elementos del código HTML. 
 
● RA7. e) Se realizarán conexiones asíncronas

(al menos 2) al servidor usando fetch. Se 
tendrá que enviar y recibir datos del servidor en 
formato JSON. 
 
Se usarán módulos para organizar el código en 
librerías (p.ej. clases) y web workers para 
realizar tareas asíncronas. 
 
Uso de variables de almacenamiento web 
local, de sesión y cookies para almacenar los 
datos de la sesión actual o de otras sesiones 
del usuario que se consideren oportunos. 
 
Se hará un control de requerimientos y filtrado 
de errores de los distintos campos. Para esto se 
usarán expresiones regulares cuando así se 
requiera. 
 
 
Resultado/s de Aprendizaje y 
Criterios de Evaluación 
Este proyecto cubre todos los RA y criterios de 
evaluación desde el RA2 - RA7. 
 
Para calcular la nota de este proyecto con respecto a 
los RA desarrollados en el centro, se tendrá en cuenta 
los requisitos implementados de cada RA y tendrán 
un peso del 10% del peso total de cada RA asociado. 
Evaluación 
En la empresa valorarán el desarrollo de este 
proyecto como “Superado o No Superado”. 
 
La nota numérica la pondrá el profesor/a titular del 
módulo de DWEC. Esta nota, si el tutor en centro de 
formación la ha puesto como “Superado”, tendrá un 
valor de 5.00 puntos sobre el 10% del valor total. La 
nota entre 5 y 10 se obtendrá mediante defensa con el 
profesor/a del proyecto para comprobar si se cubren 
los requisitos exigidos.

Introducción y 
Objetivos
Efficium Peak es una plataforma web SaaS Full-Stack orientada a empresas 
para mejorar el rendimiento, la organización y los hábitos de los empleados 
mediante microhábitos, retos y misiones, con paneles diferenciados por rol.
Objetivo General
Desarrollar una app Full-Stack con paneles diferenciados para empleados, 
managers y administradores según el plan contratado.
Competencias Aplicadas
Arquitectura web completa, metodologías ágiles, Figma, documentación 
técnica y accesibilidad WCAG.

El Problema y la Solución
El Problema
Las empresas carecen de hábitos sólidos y medibles
Los managers no disponen de datos claros de progreso
Los empleados no tienen herramientas de crecimiento 
estructurado
Imposibilidad de medir el rendimiento real del equipo
La Solución
Microhábitos diarios, retos semanales y misiones mensuales
Panel de control para managers con métricas reales
Gamificación: rachas, niveles y logros
Retos colaborativos entre equipos y demo gratuita
